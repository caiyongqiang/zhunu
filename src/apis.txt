测试 api地址:https://api.chinamall66.com/
api请求类型:全部为post
接口地址规则大致形式为:
/[接口归属类型]/[接口操作]  创建:create  更新:update  列表:list(分页字段 limit:一页多少条  offset:当前页)
以s后缀结尾的字段 传入数组时需要将数组转换成以,分隔的字符串 例如 tagIds="1,2,3"
字段数据类型默认为string 如果特殊 单独标注r
字段以is开头 数据类型为BOOLEAN  如:isBlockedByIp

status:0成功 1:失败
特殊:user/create
status:1新注册 2:旧用户 3:被拉黑

接口如果返回了errorMessage字段 直接在前端展示报错信息

//用户
/user/create  update  list
字段:
id mobilePhone password userName sex(两个值:男 女) birthday
location(设备位置信息,解析之后的城市  尽量获取精确的)
roles(角色 app传文本)
tagIds(标签,传入形式"1,2,3" 以逗号分隔)
avatar(传上传文件后接口返回的文件名)
deviceId(安卓 imei 苹果idfa)
deviceOS(设备系统)
deviceModel(手机型号)
channelId(推荐渠道id INTEGER)
registerIp(用户注册ip)
isBlockedByIp
isBlockedByDeviceId
isBlockedByMobilePhone

//管理员
/admin/login
字段:
id
userName
password
token

//标签
/tag/create  update  list
字段:
id
content(文本内容)

//通讯录
/addressBook/create  list
字段:
id
userName
mobilePhone
userId


//视频文字素材
/textStock/create  update  list
字段:
id
content(文本内容)
isRun(是否运行)
timeOut(视频最长认证时间 单位秒 INTEGER)
weight(权重 INTEGER)
tagIds


//文件上传
/upload/video image
字段:
为video时,需要同时传入userId textStockId
文件上传的统一字段为files


//渠道
/channel/create update list updatePv updateUv updateCount(后缀带有Count的字段用这个接口)
字段:
name(渠道名称)
pv(独立访客)
uv(点击量)
iosInstallCount(ios安装量)
androidInstallCount(安卓安装量)
registerCount(注册量)
inputCount(进件量  到达视频认证声明即算一个进件量)

//基本配置信息
/config/query update
字段:
maxVideos 用户最大上传视频数量
testCountBase 视频验证显示的虚拟认证基础人数
testCountEveryIncrease 每次请求新增的人数
testCountEveryIncreaseClient app端每次视频切换新增的人数
h5Url h5域名
imageUrl 图片获取地址
videoUrl 视频获取地址
